HTML, CSS, DOM and Box Model

  • Can you think of an analogy to describe HTML, CSS and the DOM and how would you describe it to your non-technical friend?

    • From the the first Udacity video they used a house to describe this. They said that you could think of the structure of a house as the HTML for example the walls and the styling of the structure as CSS for example what colour are the walls. The HTML for a web page can be thought of as the structure of it terms of what content goes on the page and the CSS is the styling of the web page in terms of colours, styling of text and how things are positioned. The DOM can be thought of in terms of the structure of a tree, where a tree has bark on the outside and it has rings going into the centre. The DOM in computing terms is the logical structure of how files and folders are arranged for a website and the way that these files can be accessed and changed if needed. The Developer Tools for a web page, show how the Document Object Model is broken down and how elements interact on the page.

  • What is meant by boxifying design?

    • Boxifying design relates to how everything that is displayed on a webpage can be thought of as fitting into a box. For example text in a paragraph, images and links can all be fitted into a box.

  • What is the box model?

    • The box model relates to how a whole web page consists of different sized boxes to make up the whole web page.
