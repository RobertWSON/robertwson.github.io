Problem Solving

Tell your non-tech friend a story about a time you:

  1. Got blocked on a simple problem
  2. Solved a problem in an elegant way in both cases:
  • What was the problem?

    • The problem was getting my website up on to GitHub for the first time.

  • What problem solving techniques did you use?

    • I came into campus, in that hope I was able to talk to someone else in the course about it. Bradley was able to point me in the right direction and tell me that you have to type your username and then github.io to give the web address after you have done git add, commit and push.

  • How did you feel throughout the process?

    • At first before I went into campus, I knew about the git add, commit and push, but was a bit confused as to how the web address was meant to come up on GitHub. I was thinking that you have the files in the repository, but how are you meant to view it as a web page. When Bradley confirmed how to do it, I felt very relaxed afterwards with the process.

  • What did you learn?

    • I learnt that I interpreted the git add, commit and push process very well. However, when I read through my notes I must of missed the last step in the process where you have to put your username in before github.io. When you are reading through notes, don't miss anything that's important.

Reflect on how confident you feel using the problem solving techniques and process:
